How it works

1) Engine Prompt

The runtime “GM brain.” Defines rules, tone, dice handling, scene structure, stat systems (e.g., Realist Kingdom System), and the Scene Integrity System that prevents drift.

// Engine Prompt — excerpt (redacted) // Defines: Roles, stats, headers, autosave hooks, safety rails // Example header enforced each scene: Scene State: { location, time, weather, present NPCs, privacy } Rules: { dice ops, XP, morality, relationship deltas }
2) Auxiliary World Files

Canonical lore packs (nations, religions, geography, factions, NPC dossiers). Loaded before play so the model doesn’t invent or drift. Multiple tiers keep big worlds tidy.

// Auxiliary (Tier I) — Cosmology & Structure Planet: Arden Fell — 32h day, 406‑day year, Aetherium veins Seas: Great Deep, Sea of Glass, Veiled Sea Climate: Equatorial archipelagos; polar aetheric auroras
3) Save / Load Chain

Per‑session state snapshots: inventory, timers, flags, relationships, kingdom stats. Autosave at end‑of‑scene; manual saves on demand. Load restores canonical state.

// Save File — minimal example Datetime: 2025‑10‑25 14:00 CDT Party: Ethan (+2), Liora (+1) Kingdom: Economy +1, Morale +2 Flags: { caravan_marked: true, demon_general: at_large } Notes: Next scene opens at city gate.

Docs (Quick)

File order discipline

Always load in this order: Engine → Save(s) → Auxiliary. For long campaigns, mount saves and auxiliaries one at a time to stay inside context limits.

Tip: Use short, distinct section headers inside each file so you can reference them by name during play (“see: Aux‑Tier‑I, Climate Bands”).
Continuity & Stability

The Scene Integrity System enforces a consistent header, dice transparency on request, and post‑scene autosave prompts. Manual saves can be triggered with a keyword.

Private modules (licensed) add kingdom economics, faction reputation matrices, and conflict clocks.
